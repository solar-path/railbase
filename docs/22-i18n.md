# 22 — Internationalization (i18n)

i18n full-stack — не только mailer templates. Распространяется на: server-side error messages, validation messages, system emails, translatable record fields, admin UI, frontend SDK.

См. также: [09-mailer.md](09-mailer.md#internationalization) (mailer-specific), [11-frontend-sdk.md](11-frontend-sdk.md) (client-side helpers).

## Locale resolution chain

При каждом request:

1. `user.language` (из record auth-collection, если задан)
2. `Accept-Language` header
3. Tenant default (multi-tenant; через `railbase-orgs` settings)
4. Site default (admin UI Settings)
5. `en` fallback

Resolved locale хранится в `context.Context` для downstream использования.

## Translation files

```
pb_data/i18n/
  en.json                              # site-wide
  ru.json
  fr.json
  de.json
  ...

pb_data/i18n/admin/                    # admin UI (отдельные ключи)
  en.json
  ru.json

tenants/{tenant_id}/i18n/              # per-tenant overrides (multi-tenant)
  en.json
  ru.json
```

Format — flat key→string maps:

```json
{
  "errors.required": "Field {field} is required",
  "errors.email_invalid": "Email format is invalid",
  "auth.signin.title": "Sign in",
  "auth.password_too_short": "Password must be at least {min} characters"
}
```

Hot-reload через fsnotify при изменении файлов.

## Schema-level translatable fields

```go
schema.Field("title", schema.Text().Translatable())
schema.Field("description", schema.RichText().Translatable())
```

Storage: separate `_translations` table:

```
_translations(record_id, collection, field, lang, value)
```

Не bloat'им main records. Read query expand'ит current locale value автоматически.

API:

```
GET /api/collections/posts/records/abc      # current locale auto-applied
GET /api/collections/posts/records/abc?lang=ru
GET /api/collections/posts/records/abc?lang=*  # all locales (admin UI editing)
```

Response (current locale):
```json
{ "id": "abc", "title": "Привет мир", "description": "..." }
```

Response (`lang=*`):
```json
{
  "id": "abc",
  "title": { "en": "Hello world", "ru": "Привет мир" },
  "description": { "en": "...", "ru": "..." }
}
```

## Server-side `t(key, params)` helper

```go
import "github.com/railbase/railbase/pkg/railbase/i18n"

msg := i18n.T(ctx, "errors.required", map[string]any{"field": "email"})
// → "Field email is required" (если en) или "Поле email обязательно" (если ru)
```

Используется для:
- Validation error messages
- Auth flow messages
- System notifications
- Built-in mailer templates (system flows: signup verify, etc.)

## JS hooks `$t()` helper

```js
throw new BadRequestError($t("errors.field_required", { field: "title" }))
```

## Frontend SDK

Client SDK получает translation bundle:

```
GET /api/i18n/{lang}              → flat key/value map для этого языка
GET /api/i18n/{lang}?prefix=auth  → только auth.* keys
```

```ts
import { createI18n } from "@railbase/sdk/i18n"

const i18n = createI18n({ baseURL, locale: "ru" })
await i18n.load()                 // fetches bundle

i18n.t("auth.signin.title")       // "Войти"
i18n.t("errors.password_too_short", { min: 8 })
                                   // "Пароль должен быть не менее 8 символов"

i18n.changeLocale("en")           // re-fetches bundle, emits event
```

## Date / number / currency formatting

Через Go `golang.org/x/text` (server-side) или browser `Intl` API (client-side).

```ts
i18n.formatDate(new Date(), "long")
// → "9 мая 2026 г." (ru) | "May 9, 2026" (en) | "9 mai 2026" (fr)

i18n.formatNumber(1234.56)
// → "1 234,56" (ru) | "1,234.56" (en)

currency.format({ amount: "1234.56", currency: "USD" }, { locale: "ru-RU" })
// → "1 234,56 $"
```

См. [11-frontend-sdk.md](11-frontend-sdk.md#domain-specific-types) для currency formatting.

## Pluralization (ICU MessageFormat-lite)

```json
{
  "comments.count": "{count, plural, =0 {No comments} =1 {1 comment} other {# comments}}"
}
```

Server-side:
```go
i18n.T(ctx, "comments.count", map[string]any{"count": 5})
// → "5 comments"
```

Client-side:
```ts
i18n.t("comments.count", { count: 5 })
```

Library: `nicksnyder/go-i18n/v2` (server) или custom mini-parser; `intl-messageformat` (client).

## RTL support

- `i18n.dir(locale)` returns `"ltr"` или `"rtl"` (Arabic, Hebrew, Urdu, Farsi)
- Admin UI auto-applies `dir="rtl"` на html element для RTL locales
- Tailwind `rtl:` variants
- Frontend SDK exposes `i18n.dir` для пользовательских apps

## Admin UI: Translations editor

Screen в admin UI (см. [12-admin-ui.md](12-admin-ui.md)):

- **Per-locale coverage %** (e.g. «en: 100%, ru: 87%, fr: 23%»)
- **Missing keys highlight** — что переведено в `en` но missing в `ru`
- **Inline editor** для каждого ключа per-language
- **Plural variant editor** для `{count, plural, ...}` keys
- **Search** по key / value
- **Bulk import/export** CSV / JSON
- **Auto-translate** через DeepL/Google Translate API (opt-in plugin `railbase-translate`)
- **Translatable record fields**: список collections с translatable fields, per-record completion status

## Built-in keys

Railbase ships с pre-translated bundles для common operations:

- Auth flows (signin, signup, reset, etc.)
- Validation errors (required, format, length, range)
- System notifications (built-in flows)
- Common admin UI labels

Languages shipped in v1: English, Russian. Community-contributed: French, German, Spanish, Chinese, Japanese, Portuguese, Italian, Polish, Turkish, Arabic.

## Per-tenant locale defaults

С `railbase-orgs` plugin: каждый tenant имеет default locale; tenant users default к нему если `user.language` not set.

## Что НЕ делает

- Sentence-level translation memory (TM) — это территория Lokalise/Crowdin/Phrase
- Auto-translation через ML — opt-in plugin `railbase-translate` (DeepL / Google / Anthropic)
- Translation workflows (review/approval) — workflow plugin может pипить
- Right-to-left admin UI редактирование контента (RTL display ok, но translatable fields в admin UI могут быть LTR-input с RTL preview)

## Open questions

- **Translation file format**: flat JSON (выбрано) vs nested vs YAML vs gettext .po? Flat — simplest, LLM-friendly.
- **Plural rules полнота**: ICU CLDR имеет up to 6 plural forms (Arabic, Russian); custom helper или off-the-shelf library.
- **Locale negotiation strict vs loose**: `pt-BR` requested, only `pt` available — fall back?
- **Translatable rich-text** — RichText fields с per-locale HTML — потенциально heavy storage.
